instant3Dhub connects to your existing PLM, PDM, and cloud storage systems through the Data Gateway without requiring any agent software on the backend side. Threedy provides reference implementations — pre-built gateway configurations and lightweight adapter services — for the most common enterprise backends. This page describes the connection pattern and key configuration points for each supported system.

</Note>

  <Tab title="Siemens Teamcenter">
    ## Siemens Teamcenter

    The Teamcenter reference implementation connects instant3Dhub to Teamcenter's REST API (Teamcenter Services REST Interface, TCCS REST). The Data Gateway calls the Teamcenter REST API to look up item revisions and retrieve associated JT or PLMXML data.

    ### Authentication

    instant3Dhub supports two authentication modes for Teamcenter:

    - **User credential pass-through** — the end user's Teamcenter session token is forwarded to the Data Gateway, ensuring that data access respects Teamcenter's own access control rules.
    - **Service account** — a dedicated Teamcenter service account authenticates on behalf of all users. Use this mode when you want to centralize access control at the instant3Dhub layer.

    The Threedy Teamcenter adapter handles session management and credential injection. Store your Teamcenter service account credentials as Kubernetes Secrets and reference them via environment variables in the adapter configuration. The reference implementation package from the support portal includes ready-to-use configuration snippets for both authentication modes.

    ### URN Mapping Pattern

    Define the URN prefix for Teamcenter items in your `values.yaml`. Teamcenter items map naturally to URNs using the item ID and revision:

    ```yaml
    urnMapper:
      mappings:
        - urnPrefix: "urn:tc:item:"
          baseUrl: "https://teamcenter.example.com/tc/api/3d/"
    ```

    With this mapping, a URN like `urn:tc:item:000123/A` resolves to the JT geometry for item 000123 revision A at the configured base URL.

    ### Reference Implementation

    The Threedy Teamcenter adapter is a lightweight Node.js service that sits between the Data Gateway and Teamcenter, handling session management, format negotiation, and JT streaming. Request the deployment package from [support.threedy.io](https://support.threedy.io).

  <Tab title="PTC Windchill">
    ## PTC Windchill

    The Windchill reference implementation connects instant3Dhub to Windchill's HTTP interface, exposing CAD documents stored in Windchill part and document structures.

    ### Authentication

    Windchill uses Basic Authentication or OAuth 2.0 (Windchill+ and later). The Threedy Windchill adapter handles credential exchange and token refresh automatically. Store your Windchill credentials as Kubernetes Secrets and reference them via environment variables. The reference implementation package from the support portal includes pre-configured examples for both Basic Auth and OAuth 2.0 modes.

    ### URN Mapping for Windchill OIDs

    Windchill identifies CAD documents by Object Identifier (OID). Define your Windchill prefix in `values.yaml`:

    ```yaml
    urnMapper:
      mappings:
        - urnPrefix: "urn:wc:part:"
          baseUrl: "https://windchill.example.com/Windchill/servlet/3d/"
    ```

    For example, `urn:wc:part:OR:wt.part.WTPart:2048431` maps to the 3D representation of that Windchill part at the configured base URL.

    ### CAD Document Access

    The Windchill adapter retrieves the latest checked-in representation for a given OID. You can configure the adapter to prefer specific format representations (e.g., PVS, STEP, JT) by adjusting the format priority list in the adapter configuration.

    Request the Windchill reference implementation package from [support.threedy.io](https://support.threedy.io).
  </Tab>

  <Tab title="Azure Blob Store">
    ## Azure Blob Store

    Connect instant3Dhub directly to an Azure Blob Storage container. This is the simplest backend integration: your 3D files live in a container, and the Data Gateway maps URNs to blob paths.

    ### Authentication

    Two authentication methods are supported for Azure Blob Storage:

    - **SAS Token** — generate a Shared Access Signature with read permissions on the container and configure it as a credential in the adapter.
    - **Service Principal** — authenticate via Azure AD using a service principal. The adapter obtains a bearer token from the Azure token endpoint and attaches it to each blob request.

    Store your Azure credentials as Kubernetes Secrets. The reference implementation package includes configuration examples for both methods.

    ### URN Mapping and Container Structure

    Define your Azure Blob Storage prefix in `values.yaml`:

    ```yaml
    urnMapper:
      mappings:
        - urnPrefix: "urn:azure:models:"
          baseUrl: "https://youraccount.blob.core.windows.net/3dmodels/"
    ```

    Organize your blob container so that blob paths map cleanly to URN suffixes. A flat structure works well for small libraries:

    ```
    3dmodels/
      assembly-4711.jt
      assembly-4712.jt
      part-001.jt
    ```

    With the mapping above, `urn:azure:models:assembly-4711.jt` resolves to `https://youraccount.blob.core.windows.net/3dmodels/assembly-4711.jt`.

    <Tip>
      Use Azure Blob Storage lifecycle management policies to automatically tier infrequently accessed files to cool or archive storage, reducing costs for large model libraries.
    </Tip>
  </Tab>

  <Tab title="Microsoft SharePoint">
    ## Microsoft SharePoint

    Use a SharePoint document library as a 3D data source. instant3Dhub accesses SharePoint files through the Microsoft Graph API, so no on-premises gateway software is required.

    ### Authentication via Microsoft Graph API

    Register an Azure AD application with the following Microsoft Graph API permissions:

    - `Sites.Read.All` — read access to SharePoint sites
    - `Files.Read.All` — read access to files in document libraries

    The adapter uses OAuth 2.0 client credentials flow to obtain tokens from the Azure AD token endpoint on behalf of the hub. Store your Azure AD application credentials as Kubernetes Secrets. The reference implementation package includes ready-to-use configuration for token acquisition and refresh.

    ### Folder Structure Mapping

    Define your SharePoint URN prefix in `values.yaml`, pointing the base URL at the relevant Microsoft Graph endpoint for your site and drive:

    ```yaml
    urnMapper:
      mappings:
        - urnPrefix: "urn:sp:models:"
          baseUrl: "https://graph.microsoft.com/v1.0/sites/{siteId}/drives/{driveId}/root:/"
    ```

    URN suffixes map to SharePoint folder paths within the document library. For example:

    ```
    urn:sp:models:Assemblies/Engine/v3.jt
              ↓
    https://graph.microsoft.com/v1.0/sites/{siteId}/drives/{driveId}/root:/Assemblies/Engine/v3.jt:/content
    ```

    The adapter appends `:/content` to the resolved path to trigger the Graph API's download redirect.

    <Note>
      SharePoint applies its own permission model. The Azure AD application you register must be granted access to the relevant SharePoint site by a SharePoint administrator before the Data Gateway can retrieve files.
    </Note>
